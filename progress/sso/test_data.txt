Here are realistic test data examples for adding SSO providers to your API, covering the most common authentication providers:
1. Azure AD (Microsoft Entra ID) - OAuth2
HTTP Request

text
POST /api/ssoproviders
Content-Type: application/json
Authorization: Bearer YOUR_ADMIN_JWT_TOKEN

{
  "providerName": "AzureAD",
  "providerType": "OAuth2",
  "clientId": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
  "clientSecret": "FyX7Q~DuPJ3kLmN8pQrStUvWxYz.AbCdEfGhIjKl",
  "authority": "https://login.microsoftonline.com/contoso.onmicrosoft.com",
  "grantType": "authorization_code",
  "scope": "openid profile email User.Read",
  "redirectUri": "https://yourapp.com/api/authentication/callback",
  "metadataUrl": "https://login.microsoftonline.com/contoso.onmicrosoft.com/v2.0/.well-known/openid-configuration",
  "isActive": true,
  "additionalSettings": {
    "tenant": "contoso.onmicrosoft.com",
    "responseType": "code",
    "responseMode": "query"
  }
}

cURL Command

bash
curl -X POST https://localhost:5001/api/ssoproviders \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ADMIN_JWT_TOKEN" \
  -d '{
    "providerName": "AzureAD",
    "providerType": "OAuth2",
    "clientId": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
    "clientSecret": "FyX7Q~DuPJ3kLmN8pQrStUvWxYz.AbCdEfGhIjKl",
    "authority": "https://login.microsoftonline.com/contoso.onmicrosoft.com",
    "grantType": "authorization_code",
    "scope": "openid profile email User.Read",
    "redirectUri": "https://yourapp.com/api/authentication/callback",
    "isActive": true
  }'

text

text
